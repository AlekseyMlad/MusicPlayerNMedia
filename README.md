# Музыкальный плеер для Android
Оригинальное задание Netology: проект профессии «Android-разработчик»

## Описание задания
Разработка нативного Android-приложения "Музыкальный плеер". Приложение получает данные об альбоме и его треках из удаленного JSON-файла, отображает список треков и предоставляет пользователю базовые функции управления воспроизведением: плей/пауза, переход к следующему/предыдущему треку и перемотка.

## Мое решение
Проект представляет собой нативное Android-приложение, написанное на Kotlin, которое реализует функциональность простого музыкального плеера.

### Архитектура:

*   **Model-View-ViewModel (MVVM):** Приложение построено на основе архитектурного паттерна MVVM.
    *   **View:** `MainActivity` отвечает за отображение пользовательского интерфейса, списка треков и элементов управления плеером. Для привязки View к коду используется `ViewBinding`.
    *   **ViewModel:** `TrackViewModel` служит для управления состоянием UI, переживает изменения конфигурации и содержит всю логику, связанную с обработкой данных (загрузка информации об альбоме, взаимодействие с плеером).
    *   **Model:** Слой данных представлен DTO-классами (`Album`, `Track`), репозиторием, который загружает данные из сети с помощью `OkHttp` и парсит их с помощью `Gson`, а также классом `MusicPlayer`, инкапсулирующим логику работы с `MediaPlayer`.
*   **Асинхронность:** Для сетевых запросов используется `OkHttp`, а для фоновых задач (получение длительности треков) применяются корутины (`Kotlin Coroutines`). Состояние плеера и данные альбома передаются в UI с помощью `LiveData`.
*   **Пользовательский интерфейс:** UI состоит из одного экрана, на котором отображается информация об альбоме, RecyclerView для списка треков и элементы управления воспроизведением (Play/Pause, Next, Previous, SeekBar).

### Основные функции:

*   **Загрузка плейлиста:** При запуске приложение асинхронно загружает информацию об альбоме и список треков из удаленного JSON-файла.
*   **Управление воспроизведением:** Пользователь может запускать и останавливать воспроизведение, а также переключаться между треками.
*   **Индикация состояния:** В списке треков подсвечивается текущий воспроизводимый трек. Кнопка Play/Pause меняет свое состояние в зависимости от того, играет музыка или нет.
*   **Прогресс трека:** `SeekBar` отображает и позволяет управлять прогрессом воспроизведения текущего трека.
*   **Обработка ошибок:** Приложение уведомляет пользователя с помощью Toast-сообщения в случае ошибки загрузки данных.

### Используемые технологии:

*   **Язык:** Kotlin
*   **Архитектура:** MVVM
*   **UI:** Android UI Toolkit, ViewBinding, RecyclerView, Material Components
*   **Асинхронность:** Kotlin Coroutines, LiveData
*   **Сеть:** OkHttp, Gson
*   **Медиа:** MediaPlayer API
*   **Сборка:** Gradle

## Инструкции по запуску
Для запуска проекта убедитесь, что у вас установлена последняя стабильная версия Android Studio.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/Zadrot-god/MusicPlayerNMedia.git
    ```
2.  **Откройте проект в Android Studio:**
    *   Запустите Android Studio.
    *   Выберите "Open an existing project".
    *   Перейдите в директорию, куда вы клонировали репозиторий, и выберите ее.
3.  **Синхронизируйте проект:**
    Android Studio автоматически начнет процесс синхронизации Gradle и загрузки всех необходимых зависимостей. Дождитесь его завершения.
4.  **Запустите приложение:**
    *   Выберите целевое устройство (эмулятор или физическое устройство).
    *   Нажмите на кнопку **Run 'app'** (зеленый треугольник) на панели инструментов.
